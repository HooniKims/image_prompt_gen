# Chrome Image Prompt Assistant 최종 스펙

작성일: 2026-05-25

## 1. 목적

Chrome Image Prompt Assistant는 ChatGPT와 Gemini 웹페이지에서 바로 사용할 수 있는 이미지 프롬프트 작성 보조 확장 프로그램입니다.

사용자는 한국어로 주제와 조건을 입력하고, 확장은 이미지 생성 모델에 넣기 좋은 영어 중심 프롬프트를 만들어 현재 입력창에 삽입합니다. 교육용 이미지, 스토리북, 만화/웹툰, 여러 장 시리즈, 안내 이미지처럼 반복적으로 쓰는 작업 흐름을 빠르게 처리하는 것이 핵심입니다.

## 2. 최종 구성

- Chrome Manifest V3 확장 프로그램
- ChatGPT / Gemini 페이지 자동 감지
- 감지된 사이트에 맞춰 `대상` 선택값 자동 동기화
- Shadow DOM 기반 오른쪽 사이드 패널
- 초기 페이지 shell이 늦게 준비되거나 mount 중 오류가 나도 패널 주입을 재시도
- Mac mini Prompt API 호출
- LM Studio OpenAI-compatible API 연동
- API 호출은 background service worker에서 수행
- API 장애 시 조용한 브라우저 fallback 대신 연결 실패 표시
- 최근 작업 / 즐겨찾기 / 버전 관리
- 49개 스타일별 자체 생성 WebP 스타일 썸네일과 사용자 직접 입력 스타일
- image-gen으로 생성한 확장 프로그램 아이콘 16/32/48/128 PNG
- 최종 배포 ZIP 생성 스크립트

## 3. 배포 파일

최신 배포 산출물은 아래 한 파일입니다.

- `release/chrome-image-prompt-assistant.zip`

예전에는 같은 내용을 두 ZIP 이름으로 만들었지만, 현재는 중복을 줄이기 위해 `chrome-image-prompt-assistant.zip`만 생성합니다.

## 4. 사용자 화면 기능

### 기본 입력

- 주제 입력
- 용도 선택
- 스타일 선택
- 화면 비율 선택
- 품질 선택
- 이미지 안 문구 입력
- 대상 사이트 자동 감지 또는 선택

### 작업 유형

기본 대표 작업과 더보기 작업 유형을 제공합니다.

- 일반 이미지
- 교육 설명 이미지
- 수업자료 슬라이드
- 카드뉴스 / 에디토리얼
- 비교 / 컨설팅 자료
- 스토리북
- 만화 / 웹툰
- 여러 장 시리즈
- 유튜브 썸네일
- 인스타그램 피드 / 스토리
- A4 문서
- 블로그 대표 이미지
- 학생 / 학부모 안내

이전에는 `작업 유형`과 `출력 형식`을 따로 선택했지만, 현재는 사용자 혼동을 줄이기 위해 화면에서 `출력 형식` 선택을 제거했습니다. 작업 유형이 결과물의 성격과 출력 형식을 함께 결정합니다. 예를 들어 `스토리북`을 작업 유형으로 선택하면 내부 요청의 `formatPreset`도 `storybook`으로 자동 설정되고, `수업자료 슬라이드`를 선택하면 `class_slide`로 자동 설정됩니다.

작업 유형은 단순한 라벨이 아니라 프롬프트 생성 규칙도 함께 결정합니다. 카드뉴스/인포그래픽은 제목, 핵심 메시지, 3-5개 정보 블록, 아이콘/도식, 짧은 텍스트, 명확한 정보 위계를 우선합니다. 유튜브 썸네일은 강한 중심 피사체, 고대비, 큰 제목 영역, 모바일 가독성, 과밀 방지를 우선합니다. 수업자료 슬라이드는 학습 목표, 교실에서 크게 보이는 개념도, 라벨/화살표, 투사 가독성, 장식 최소화를 우선합니다. A4 문서는 인쇄용 세로 레이아웃, 충분한 여백, 제목/섹션 박스, 표나 체크리스트, 종이 출력 가독성을 우선합니다. 블로그 대표 이미지, SNS/블로그 이미지, 학생/학부모 안내, 교재/활동지 삽화, 캐릭터/배경/소품도 각각의 목적에 맞는 세부 규칙을 API 요청에 자동으로 포함합니다.

고급 옵션의 `추가 조건`은 이 기본 작업 유형 규칙을 더 구체화하거나 명시적으로 덮어쓰는 용도입니다. 예를 들어 유튜브 썸네일에서 기본적으로 고대비를 권장하더라도, 사용자가 `차분한 색감, 과한 표정 금지`라고 쓰면 이 요청을 우선해 반영하도록 system prompt에 지시합니다.

### 스토리북 / 만화 / 여러 장 워크플로우

스토리북, 만화, 여러 장 시리즈처럼 여러 이미지를 만들어야 하는 경우에는 단일 프롬프트가 아니라 번호가 붙은 prompt pack을 생성합니다.

생성 장수는 1장부터 12장까지 1장 단위로 선택할 수 있습니다. 장면 또는 컷 구성이 비어 있어도 생성 장수가 2장 이상이면 선택한 장수와 정확히 같은 개수의 numbered prompt pack을 만들도록 지시합니다. 이때 API 요청에는 `No panel/scene plan was provided, so split the source idea into exactly N natural pages, panels, or scenes yourself.` 지시를 포함해 모델이 원본 아이디어를 자연스러운 장면, 컷, 페이지로 직접 나누게 합니다.

API 모델이 최종 응답에서 생성 장수 조건을 빠뜨린 경우에도 확장 프로그램은 정규화 단계에서 `Return a numbered prompt pack with exactly N image prompts.` 문장을 최종 프롬프트에 보강합니다. 따라서 사용자가 2장 이상을 선택하면 자동 입력되는 프롬프트 본문에도 생성 장수 조건이 남아 있어야 합니다.

스토리북과 만화/웹툰은 장르 특성상 사용자가 별도로 요청하지 않아도 짧은 대사나 말풍선 문구를 기본 포함하도록 지시합니다. 스토리북은 페이지별 1-2개의 짧은 대사 또는 캡션을, 만화/웹툰은 컷별 1-2개의 간결한 말풍선을 기본값으로 사용합니다. 단, 사용자가 `대사 없음`, `말풍선 없음`, `텍스트 최소화`, `no dialogue`, `minimal text`처럼 텍스트를 줄이거나 없애라고 명시하면 이 기본 대사 지시는 넣지 않습니다.

폼에서 받을 수 있는 주요 조건은 아래와 같습니다.

- 생성 장수
- 장면 또는 컷 구성
- 캐릭터 일관성 조건
- 배경 일관성 조건
- 반복 소품 / 색상 / 분위기

## 5. 스타일 시스템

처음에는 49개 스타일을 그대로 보여주는 방식이었지만, 이후 비슷한 스타일을 묶어 8개 카테고리로 정리했습니다. 현재 구현은 카테고리 구조는 유지하되, 각 스타일마다 서로 다른 자체 생성 WebP 썸네일을 사용합니다.

기본 화면에는 각 카테고리 대표 스타일 1개씩만 보여줍니다. 전체 보기를 누르면 전체 49개 스타일 목록을 볼 수 있습니다. 각 스타일 카드는 `assets/style-samples/style-01.webp`부터 `style-49.webp`까지의 개별 썸네일을 사용해, 영어 설명을 읽지 않아도 이미지 자체만으로 스타일 차이를 빠르게 파악할 수 있게 합니다. 썸네일은 UI 슬롯과 같은 가로형 16:9 구도로 직접 생성하고, 1:1 이미지를 넣은 뒤 남는 공간을 블러 처리하는 방식은 사용하지 않습니다. 칠판 스타일은 초록색 칠판과 흰 분필 선으로 표현합니다.

스타일 선택 입력칸에는 내부 작업명인 `Notion LM 50 스타일 보기`를 표시하지 않습니다. 선택된 스타일명과 목록을 여는 화살표만 보여 주어, 다른 선택형 입력과 같은 방식으로 이해되게 합니다.

스타일 선택 패널 상단에는 `직접 입력 스타일` 입력칸을 둡니다. 사용자는 49개 프리셋 에셋을 고를 수도 있고, `어두운 SF 영화 포스터, 파란 네온 조명, 강한 대비`처럼 원하는 스타일을 직접 써서 적용할 수도 있습니다. 직접 입력한 스타일은 프리셋과 같은 `Style:` 값으로 API 요청에 전달됩니다.

8개 카테고리는 아래와 같습니다.

- 웹/UI·SaaS
- 교육·설명
- 카드뉴스·에디토리얼
- 데이터·비교·컨설팅
- 일러스트·공예
- 레트로·코믹·팝
- 3D·아이소메트릭
- 시네마틱·미래형

권리 문제를 피하기 위해 외부 Notion 원본 이미지는 최종 ZIP에서 제거했습니다. 대신 image-gen으로 만든 49개 스타일별 자체 썸네일을 320x180 WebP로 압축해 사용합니다. 예전 8개 카테고리 대표 썸네일은 더 이상 사용하지 않습니다.

다른 컴퓨터에서 이어서 작업할 때 컨텍스트를 복구할 수 있도록 루트에 `handoff-context.md`를 둡니다. 이 파일에는 현재 API/모델 기본값, 최근 UI 문구 결정, 스타일 썸네일 구조, 검증 명령, 새 세션 시작 문구를 정리합니다.

## 6. 프롬프트 생성 방식

프롬프트 생성의 정상 경로는 API 호출입니다. 사용자가 `프롬프트 생성`을 누르면 확장은 API 연결을 먼저 시도하며, 실패를 성공처럼 숨기지 않습니다.

### 1차 경로: Mac mini Prompt API

확장은 기본적으로 아래 API를 호출합니다.

```text
POST https://image-prompt.alluser.site/api/image-prompt
```

서버는 Mac mini에서 실행되는 Prompt API이며, LM Studio의 OpenAI-compatible API로 요청을 전달합니다.

현재 모델 순서는 아래와 같습니다.

- 기본: `google/gemma-4-e2b`
- 복잡한 요청: `google/gemma-4-e4b`

설정 패널의 모델명 입력칸은 기본값으로 `google/gemma-4-e2b`가 자동 입력됩니다.

내부 배포용 기본 API 키도 확장 프로그램 설정 기본값으로 함께 채웁니다. 사용자가 직접 저장한 API 키가 있으면 저장값을 우선하고, 저장값이 없을 때만 기본 키를 사용합니다.

고급 옵션의 입력 의미는 아래처럼 구분합니다.

- `이미지에 표시할 정확한 문구`: 제목, 말풍선, 표지 문구처럼 최종 이미지 안에 실제로 보일 텍스트입니다. 프롬프트에는 원문 보존 지시로 들어갑니다.
- `추가 조건`: 스타일, 구도, 금지 요소, 강조점처럼 프롬프트 생성에 추가로 반영할 지시입니다. 이 값은 이미지 안에 그대로 표시되는 문구가 아니라 API 요청의 추가 지시로 전달됩니다. placeholder는 `예: 배경은 밝은 교실, 텍스트는 최소화, 손가락은 정확하게, 로고/워터마크 제외`처럼 바로 따라 쓸 수 있는 예시문으로 표시합니다.
- `참고 자료`: `사용자가 채팅창에 첨부파일로 업로드함`으로 표시해, 확장 프로그램 내부 업로드가 아니라 ChatGPT/Gemini 채팅창에 첨부한 자료를 뜻한다는 점을 분명히 합니다.

동시 접속 시 e4b 부하가 커질 수 있으므로 일반 요청은 e2b로 처리합니다. 다음 조건 중 하나라도 해당하면 hosted Prompt API 요청 본문에 `google/gemma-4-e4b`를 선택 모델로 전달합니다.

- 품질이 `high`
- 생성 개수가 2개 이상
- 작업 유형이 `storybook`, `comic_webtoon`, `image_series`
- 장면/컷 구성, 일관성 조건, 이미지 안 문구가 입력됨

기본 배포판은 사용자가 LM Studio 주소를 직접 입력하지 않아도 되도록 항상 hosted Prompt API를 기본 API 주소로 사용합니다. 이전 테스트 과정에서 저장된 `lm.alluser.site`, `localhost`, `127.0.0.1` 직접 호출 주소는 확장 초기화 시 기본 Prompt API 주소로 되돌립니다.

### API 호출 위치

ChatGPT/Gemini 페이지에 주입된 content script에서 외부 API를 직접 호출하지 않습니다.

프롬프트 생성 버튼을 누르면 content script가 background service worker로 메시지를 보내고, background service worker가 `host_permissions` 권한으로 hosted Prompt API를 호출합니다.

이 구조는 ChatGPT/Gemini 페이지 origin에서 발생할 수 있는 CORS 차단을 피하기 위한 것입니다.

### API 실패 처리

Mac mini API 또는 LM Studio 호출이 실패하면 즉시 브라우저 fallback 프롬프트를 저장하지 않습니다.

대신 사용자에게 API 연결 실패 메시지를 표시하고 최근 작업에도 저장하지 않습니다. 사용자는 API 주소, API 키, LM Studio 실행 상태, 모델명을 확인한 뒤 다시 생성해야 합니다.

브라우저 fallback 생성 코드는 비상용 구현으로 남겨두지만, 기본 생성 버튼의 정상 경로에서는 조용히 사용하지 않습니다.

## 7. Image 2.0 / Nano Banana 가이드 반영

레퍼런스 기준은 현재 런타임 코드에 내장되어 있습니다. 이전에 별도 문서로 관리하던 가이드 요약은 최종 확장 프로그램 폴더 안의 필수 파일로 남기지 않고, 실행 경로에서 바로 쓰는 상수와 함수로 정리했습니다.

반영한 기준은 아래 두 출처를 요약한 것입니다.

- OpenAI Academy: Creating images with ChatGPT
- Google DeepMind: Gemini Image / Nano Banana prompt guide

중요한 점은, 프롬프트를 만들 때마다 인터넷 문서를 다시 읽는 구조가 아니라는 것입니다. 공식 문서에서 뽑은 규칙을 프로젝트 안에 넣어두고, 실제 프롬프트 생성 시 자동으로 섞어 넣는 구조입니다.

실제 코드 반영 위치는 아래입니다.

```text
src/content-runtime.js
```

핵심 함수와 상수는 아래입니다.

- `IMAGE_GUIDE_SET`
- `canonicalGuideSet()`
- `targetGuideInstructions()`
- `purposeGuidance()`
- `purposeGuidanceKo()`

이 기준은 두 곳에 모두 들어갑니다.

- LM Studio / OpenAI-compatible system prompt
- 비상용 브라우저 fallback 프롬프트

정상 생성은 API 응답 기준으로만 완료 처리합니다. API가 실패한 경우에는 사용자가 실패를 알 수 있어야 하며, 임시 fallback 결과를 성공처럼 보여주지 않습니다.

## 8. 입력창 삽입 방식

생성된 프롬프트는 현재 ChatGPT 또는 Gemini 입력창에 삽입됩니다.

확장이 주입된 페이지가 Gemini이면 `대상` 선택값도 자동으로 `Gemini용`으로 맞춥니다. 사용자가 ChatGPT/Gemini가 아닌 페이지에서 열었을 때만 수동 선택값을 유지합니다.

삽입 시 사용자는 아래 중 하나를 선택할 수 있습니다.

- 기존 내용 덮어쓰기
- 기존 내용 뒤에 이어붙이기
- 취소

Gemini 입력창은 contenteditable 기반 제어 컴포넌트이므로 단순 `textContent` 대입 후 즉시 읽으면 삽입 검증이 실패할 수 있습니다. 런타임은 focus, selection, `beforeinput`, `execCommand('selectAll')`, `insertText`, `input` 이벤트를 함께 사용하고, 삽입 후 짧게 재조회해 실제 composer 텍스트에 프롬프트가 들어갔는지 검증합니다.

## 9. 저장 기능

브라우저 `chrome.storage.local`을 사용합니다.

저장되는 항목은 아래와 같습니다.

- 최근 작업
- 즐겨찾기
- 생성 버전
- 관리자 설정

최근 작업은 최대 100개까지 유지합니다.

## 10. 관리자 설정

숨겨진 관리자 설정에서 아래 값을 조정할 수 있습니다.

- API 주소
- API 키
- 모델명

기본 API 주소는 아래입니다.

```text
https://image-prompt.alluser.site/api/image-prompt
```

기본 모델명은 아래입니다.

```text
google/gemma-4-e2b
```

hosted Prompt API 서버는 복잡한 요청 또는 고품질 요청에서 아래 모델을 사용합니다.

```text
google/gemma-4-e4b
```

API 키 입력칸은 브라우저가 비밀번호 입력으로 인식하지 않도록 일반 텍스트 입력과 `autocomplete="off"` 설정을 사용합니다.

결과 영역에서는 일반 사용자에게 내부 키워드 칩을 표시하지 않습니다. `purpose`, `formatPreset`, `description`, `targetSite` 같은 키/값 요약 대신 API가 내려주는 `promptKo` 또는 `koreanPrompt` 값을 사용해 프롬프트의 한글 설명을 보여줍니다. API가 한글 설명을 내려주지 못한 경우에는 확장 프로그램이 프롬프트의 주제, 스타일, 비율, 품질 정보를 바탕으로 읽기 쉬운 한글 안내를 생성합니다. 다시 생성하기에서 사용한 수정 요청도 한글 설명 fallback에 함께 표시해 결과가 바뀐 이유를 사용자가 확인할 수 있게 합니다. 결과 액션 버튼은 짧게 `자동 입력`으로 표시하고, `복사` 버튼은 초록색 강조 버튼으로 구분합니다.

API 응답의 개선 제안이 영어 문장으로 내려와도 결과 영역에서는 일반 사용자가 읽기 쉬운 한글 안내 문장으로 변환해 표시합니다. OpenAI-compatible 응답이 JSON이 아니라 일반 텍스트 프롬프트만 반환된 경우의 안내 문구는 내부 구현명 대신 `외부 API의 응답을 프롬프트 본문으로 사용했습니다.`로 표시합니다. 이 추가 조언은 한글 프롬프트 설명 아래에 목록으로 표시합니다.

프롬프트 한글 설명 영역에는 API가 내려준 `promptKo`가 있더라도 확장이 적용한 작업 유형별 세부 규칙을 `적용된 작업 유형 규칙:` 문장으로 덧붙입니다. 사용자는 결과 프롬프트가 어떤 목적별 기준을 반영했는지 바로 확인할 수 있습니다.

hosted Prompt API와 LM Studio 응답이 오래 걸릴 수 있으므로 확장 프로그램의 기본 API 요청 timeout은 90초로 둡니다. background fetch 응답 대기 timeout은 95초로 두어 요청 abort와 응답 대기 사이에 약간의 여유를 둡니다. timeout이 발생하면 사용자에게 `서버 응답이 오래 걸리고 있습니다. 잠시 후 다시 시도하거나 출력 개수나 품질을 낮춘 뒤 다시 시도해 주세요.`라고 안내합니다. 서버/Nginx 운영 환경에서는 `proxy_read_timeout`, `proxy_send_timeout`, API 서버 자체 timeout을 120초 이상으로 맞추는 것을 권장합니다.

`프롬프트 만들기`를 누르면 버튼 문구가 `프롬프트 생성중...`으로 바뀌고 버튼이 비활성화되어 생성 상태를 명확히 보여 줍니다. 완료/실패 알림 toast는 어두운 배경, 흰 글자, padding, shadow를 가진 눈에 띄는 박스로 표시합니다.

패널을 접었을 때 보이는 플로팅 실행 버튼은 `프롬프트`와 `만들기`를 두 줄로 나누어 표시합니다. 첫 사용 안내의 확인 버튼은 `사용방법 확인`으로 표시하고, 빨간 배경과 흰색 글자를 사용해 사용자가 먼저 눌러야 하는 CTA임을 명확히 보여 줍니다.

ChatGPT/Gemini 페이지를 새로고침할 때 사이트 자체의 앱 shell이 늦게 준비되거나 content script 초기화 중 일시 오류가 생겨도 한 번 실패하고 끝나지 않도록, 런타임은 최대 20회까지 짧은 간격으로 패널 mount를 재시도합니다. mount 중 실패한 host는 제거한 뒤 다음 시도에서 다시 붙입니다.

결과 아래의 빠른 개선 버튼은 즉시 재생성하지 않고 선택 상태만 표시합니다. 사용자가 빠른 개선 버튼 하나를 선택하거나 직접 피드백을 입력한 뒤 `다시 생성하기`를 눌러야 다시 API를 호출합니다. 피드백 입력창 placeholder는 `수정 방향을 선택하거나 직접 입력한 뒤 다시 생성하기를 누르세요`로 표시합니다. 개선 생성 중에는 피드백 입력창에 `수정사항을 반영하여 다시 프롬프트를 생성중입니다`가 무지개빛 애니메이션 상태로 표시되고, 완료 후에는 `수정이 완료되었습니다.`가 표시됩니다. 개선 생성 중에는 다른 개선 버튼과 피드백 버튼을 비활성화해 중복 요청을 막습니다.

스토리북, 만화/웹툰, 여러 장 시리즈 작업 유형은 기본 작업 유형 목록에서 바로 선택할 수 있습니다. 출력 형식은 별도 목록으로 다시 묻지 않고 작업 유형에서 자동 결정합니다.

현재 내부 배포판은 기본 API 키를 설정 기본값으로 미리 채웁니다. 사용자가 숨겨진 설정에서 직접 저장한 API 키가 있으면 저장값을 우선 사용합니다.

## 11. 개인정보 / 보안 원칙

- ChatGPT/Gemini 대화 전체를 자동으로 읽지 않습니다.
- 사용자가 입력한 주제와 조건만 프롬프트 생성에 사용합니다.
- 생성 이미지는 저장하지 않습니다.
- 참고 파일은 자동 업로드하지 않고 사용자가 직접 업로드하도록 안내합니다.
- 사용자 작업 기록은 기본적으로 브라우저 로컬 저장소에만 저장합니다.
- 내부 배포용 기본 API 키는 설정 기본값으로 포함하며, 사용자가 저장한 설정값을 우선합니다.
- API 호출 실패를 숨기고 로컬 fallback 결과를 성공처럼 저장하지 않습니다.
